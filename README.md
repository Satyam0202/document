# Personal Document Vault

A secure, single-user document management app built with Node.js + Express. Upload files directly to **Google Drive** from a clean dashboard — no third-party storage service needed.

---

## Features

- Login-protected dashboard (single-user)
- Upload documents → saved directly to your Google Drive folder
- Organize by category, search, preview, download, delete
- Dark / light theme toggle
- Fully deployable from GitHub to Render (free tier)

---

## Local Development

### 1. Clone & install

```bash
git clone https://github.com/<your-username>/<repo-name>.git
cd <repo-name>
npm install
```

### 2. Set up environment variables

```bash
copy .env.example .env
```

Edit `.env` with your values (see comments inside the file).

### 3. Set up Google Drive API

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project (or use existing)
3. Enable **Google Drive API**
4. Go to **IAM & Admin → Service Accounts** → Create service account
5. Download the JSON key → save as `gdrive-service-account.json` in the project root
6. Open your Google Drive folder → Share it with the service account email (Editor access)
7. Copy the folder ID from the URL and set `GOOGLE_DRIVE_FOLDER_ID` in `.env`

### 4. Run

```bash
npm start
```

Visit `http://localhost:3000`

## Deploy to Render (from GitHub — free, no credit card)

### Step 1 — Push to GitHub

```bash
git add .
git commit -m "initial commit"
git push origin main
```

> Make sure `.env` and `gdrive-service-account.json` are in `.gitignore` (already done).

### Step 2 — Create Render Web Service

1. Go to [render.com](https://render.com) → Sign up / Log in with GitHub
2. Click **New → Web Service**
3. Connect your GitHub repo
4. Render will auto-detect `render.yaml` — settings are pre-filled
5. Click **Create Web Service**

### Step 3 — Set Environment Variables in Render Dashboard

Go to your service → **Environment** tab and add:

| Key | Value |
|-----|-------|
| `VAULT_USERNAME` | your login username |
| `VAULT_PASSWORD` | your login password |
| `GOOGLE_DRIVE_FOLDER_ID` | your Drive folder ID |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | paste the **entire contents** of `gdrive-service-account.json` |

> `SESSION_SECRET` is auto-generated by Render (already in `render.yaml`).

### Step 4 — Deploy

Render auto-deploys on every push to `main`. Your app will be live at:

```
https://<service-name>.onrender.com
```

## How Google Drive Upload Works

1. You upload a file via the dashboard
2. The app temporarily saves it in `uploads/` (local, gitignored)
3. It uploads the file to your Drive folder using the service account
4. The temp local file is deleted
5. The document record stores the Drive `fileId`, `webViewLink`, and `webContentLink`
6. Preview → opens the file in Google Drive Viewer
7. Download → direct download link from Drive

## Environment Variables Reference

| Variable | Required | Description |
|----------|----------|-------------|
| `PORT` | No | Server port (default: 3000) |
| `NODE_ENV` | No | Set to `production` on Render |
| `SESSION_SECRET` | Yes | Random string for session signing |
| `VAULT_USERNAME` | Yes | Login username |
| `VAULT_PASSWORD` | Yes | Login password |
| `GOOGLE_DRIVE_FOLDER_ID` | Yes | Drive folder ID for uploads |
| `GOOGLE_APPLICATION_CREDENTIALS` | Local dev | Path to service account JSON file |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | Production | Full JSON content of service account |
| `UPLOAD_LIMIT_MB` | No | Max file size in MB (default: 10) |
| `ALLOWED_EXTENSIONS` | No | Comma-separated allowed extensions |

## Tech Stack

- **Backend**: Node.js, Express
- **Views**: EJS
- **Auth**: express-session (single-user)
- **Storage**: Google Drive API (googleapis)
- **File upload**: multer (temp local → Drive)
- **Deploy**: Render (free tier, GitHub integration)
